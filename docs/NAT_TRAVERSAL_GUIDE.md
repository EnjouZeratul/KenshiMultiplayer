# NAT 穿透配置指南

## 功能概述

本模组支持 NAT 穿透功能，允许互联网玩家连接到您的主机游戏，而不仅限于局域网。

### NAT 穿透优先级

系统按以下优先级尝试建立连接：

1. **UPnP (首选)** - 自动在路由器上配置端口转发
2. **STUN** - 通过公共服务器发现您的公网 IP
3. **局域网模式** - 仅本地网络可连接（ fallback）

## 作为主机

### 互联网连接（推荐）

如果您希望互联网上的玩家能够加入：

1. 确保路由器支持 UPnP 并已启用
2. 启动主机时，系统会自动尝试 UPnP 端口转发
3. 如果 UPnP 成功，其他玩家可以通过您的公网 IP 连接
4. 如果 UPnP 失败，系统会尝试 STUN 方式

### 局域网连接

如果仅在本地网络内游戏：

- 系统会自动检测并显示您的本地 IP（如 192.168.x.x）
- 其他本地网络内的玩家可以直接连接

### 获取您的连接信息

启动主机后，系统会显示：
- **本地 IP** - 用于局域网连接
- **公网 IP** - 用于互联网连接（如果 NAT 穿透成功）
- **端口** - 默认为 27960

将相应的 IP 和端口信息告知要加入的玩家。

## 作为客户端

### 连接到互联网主机

1. 选择"客户端"模式
2. 输入主机提供的公网 IP 地址
3. 输入端口（默认 27960）
4. 点击连接

### 连接到局域网主机

1. 选择"客户端"模式
2. 输入主机的本地 IP 地址
3. 输入端口
4. 点击连接

## 高级配置

### 配置 STUN 服务器

如果默认的公共 STUN 服务器不可用，可以配置自定义服务器：

编辑 `config/settings.json`，添加：

```json
{
  "nat_traversal_enabled": true,
  "nat_upnp_enabled": true,
  "nat_stun_enabled": true,
  "custom_stun_server": "stun.l.google.com:19302"
}
```

### 默认 STUN 服务器

系统内置了多个冗余 STUN 服务器：
- stun.l.google.com:19302
- stun1.l.google.com:19302
- stun.stunprotocol.org:3478
- 223.5.5.5:3478 (阿里云)
- 180.76.76.76:3478 (百度)

## 故障排除

### UPnP 失败

如果 UPnP 失败：
1. 检查路由器是否支持 UPnP
2. 在路由器设置中启用 UPnP
3. 检查防火墙是否允许 UPnP 请求

### STUN 失败

如果 STUN 失败：
1. 检查网络连接
2. 尝试更换 STUN 服务器
3. 某些 NAT 类型（如对称 NAT）可能不支持 STUN

### 仅局域网模式

如果 NAT 穿透完全失败：
- 您仍然可以在局域网内游戏
- 互联网玩家需要其他方式（如虚拟局域网工具）才能加入

## 安全说明

- 本模组不修改任何游戏文件
- 所有网络通信使用 UDP 协议
- NAT 穿透功能完全在模组内部实现
- 删除 KenshiMultiplayer 文件夹即可完全卸载
